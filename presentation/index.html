📋 方案概述

本方案是一个基于 多 Agent 协作 的全栈开发系统，采用 Spec-First（规范优先） 的开发范式，通过 Orchestrator 协调 Backend、Frontend、QA 三个专业 Agent 完成 DRF + Vue3 项目的全流程开发。

核心理念： 代码未动，规格先行。所有功能必须先在 .specify/specs/ 中定义 API 合同、UI 要求和测试用例，然后由对应 Agent 执行实现。

🎯 Spec-First

  • 规格文档先行
  • API 合同明确
  • 测试用例预定义

🧪 TDD Mandate

  • 测试先于实现
  • Red → Green → Refactor
  • 覆盖率 >80%

🔒 Type Safety

  • 类型安全合同

👥 Agent 职责详解

🎯 Orchestrator Agent

系统架构师 & 项目经理

  • 分析用户需求，检查现有 Spec
  • 缺失时协调创建规格文档
  • 分解任务为 Backend/Frontend/QA
  • 协调 API 合同分歧
  • 追踪进度，更新 progress.md
  • 处理 QA 阻塞问题

🔧 Backend Agent

DRF + TDD 专家

  • 阅读规格文档
  • 先写失败测试 (RED)
  • 实现最小代码 (GREEN)
  • 重构优化 (REFACTOR)
  • 确保覆盖率 >80%
  • 避免 N+1 查询

🎨 Frontend Agent

Vue3 + UI/UX 专家

  • 定义 TypeScript 类型
  • 创建 API 客户端
  • 实现 Pinia Store
  • 开发 Vue 组件
  • 应用 UI-UX-Pro-Max 原则
  • 确保 0 lint/type 错误

✅ QA Agent

质量门禁

  • 运行 pytest 验证测试
  • 运行 npm lint/type-check
  • 验证 API 合同匹配规格
  • 性能测试 (p95 <200ms)
  • 生成 QA 报告
  • 阻塞不达标合并

📐 Spec-First 开发模式

核心原则： 没有规格文档，就没有代码实现。Spec 是唯一事实来源。

Spec 文档结构

# 规格：[功能名称]

## Overview
功能概述和业务价值

## API Contract
- 端点定义
- 请求/响应格式
- 错误响应
- 状态码

## UI Requirements
- 布局要求
- 颜色/间距/字体
- 响应式断点
- 无障碍要求
- 状态处理 (Loading/Error/Empty)

## Test Cases
- 单元测试用例
- 集成测试用例
- 边界 case

## Implementation Checklist
- Backend 检查清单
- Frontend 检查清单

## Files to Create/Modify
- 需要创建/修改的文件列表

Spec 管理流程

📄 创建

Orchestrator 协调 Backend/Frontend/QA 共同定义 API 合同、UI 需求、测试用例

📝 更新

变更需记录理由到 decisions.md，版本号 +1，通知相关 Agent

📋 验证

QA 验证实现与 Spec 合同 100% 匹配，不匹配则阻塞

🧪 TDD 强制流程 (Backend)

RED写失败测试
→
GREEN最小实现
→
REFACTOR重构优化

阶段详解

阶段 动作 验证命令 通过标准
RED 编写失败测试 pytest tests/test_xxx.py -v 测试失败 (退出码 1)
GREEN 写最小实现代码 pytest tests/test_xxx.py -v 测试通过 (退出码 0)
REFACTOR 重构，保持测试绿 pytest -v --cov=app 测试通过 + 覆盖率>80%
"没有测试的实现就是裸奔——裸奔上线等于提前写复盘。"

代码示例

# Step 1: RED - 写失败测试 (tests/test_user.py)
def test_user_creation():
    user = User.objects.create(username='test', email='test@example.com')
    assert user.username == 'test'
    assert user.is_active == True

# Step 2: GREEN - 最小实现 (models.py)
class User(models.Model):
    username = models.CharField(max_length=150)
    email = models.EmailField()
    is_active = models.BooleanField(default=True)

# Step 3: REFACTOR - 重构优化
class User(models.Model):
    """用户模型 - 认证和授权基础。"""
    
    username = models.CharField(
        max_length=150,
        unique=True,
        db_index=True  # 添加索引
    )
    email = models.EmailField(unique=True)
    is_active = models.BooleanField(default=True)
    
    class Meta:
        db_table = 'auth_user'
        ordering = ['username']
