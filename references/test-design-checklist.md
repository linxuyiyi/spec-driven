# 测试设计检查清单

> 设计测试用例前的自检清单。颗粒度拉到这么细才叫端到端——只写步骤不检查，那叫头痛医头。

---

## 设计前：需求澄清 (TRF-T 信任)

在开始写测试步骤之前，先问自己：

- [ ] **测试目的是什么？** 验证功能正常？发现问题？回归测试？
- [ ] **验收标准是什么？** 什么叫"测试通过"？
- [ ] **前置条件有哪些？** 需要哪些环境、配置、资源？
- [ ] **被测对象是谁？** 哪个网元？哪个版本？
- [ ] **有什么约束？** 时间、资源、环境限制？

---

## 设计中：完整性检查

### 步骤完整性

- [ ] 有**环境重置**步骤吗？(清空残留配置)
- [ ] 有**前置条件**步骤吗？(小区建立、UE 附着等)
- [ ] 有**执行步骤**吗？(触发条件、动作执行)
- [ ] 有**验证步骤**吗？(至少 3 个验证点)
- [ ] 有**后置处理**吗？(环境清理、资源释放)

### 参数规范性

- [ ] 所有参数带**单位**吗？(MHz/dBm/ms 等)
- [ ] 参数值在**合理范围**内吗？
- [ ] 用户未指定的参数使用**默认值**了吗？
- [ ] 关键参数有**注释说明**吗？

### 预期结果明确性

- [ ] 每个步骤有**预期结果**吗？
- [ ] 预期结果**可量化验证**吗？
- [ ] 验证步骤的预期结果写**期望观测现象**了吗？

---

## 设计中：覆盖度检查

### 场景覆盖

- [ ] **正常流程**覆盖了吗？
- [ ] **变体场景**考虑了吗？(至少 1 个)
- [ ] **负例场景**考虑了吗？(至少 1 个)
- [ ] **边界条件**考虑了吗？(门限边缘、参数极限)

### 协议覆盖

- [ ] 涉及的**3GPP 协议**条款明确了吗？
- [ ] 协议定义的**触发条件**正确吗？
- [ ] 协议定义的**定时器/计数器**考虑了吗？

### 失败模式覆盖

- [ ] **常见失败原因**对应的验证有了吗？
- [ ] **超时处理**考虑了吗？
- [ ] **重试机制**考虑了吗？

---

## 设计中：可执行性检查

### AW 函数映射

- [ ] 每个步骤能映射到**AW 函数**吗？
- [ ] AW 函数**参数匹配**吗？
- [ ] AW 函数**返回值处理**了吗？

### 步骤粒度

- [ ] 每个步骤是**原子操作**吗？(不可再分)
- [ ] 步骤名用**动词 + 名词**格式吗？
- [ ] 步骤之间**依赖关系**清晰吗？

### 验证可行性

- [ ] 验证步骤有对应的**验证函数**吗？
- [ ] 验证**超时时间**合理吗？
- [ ] 验证**通过/失败判断**明确吗？

---

## 设计后：自检评审

### Keeper Test

- [ ] 这个测试用例值得保留吗？
- [ ] 如果同事看到这个用例，会觉得专业吗？
- [ ] 这个用例能发现真正的问题吗？

### 蓝军视角

- [ ] 最可能在哪个步骤**失败**？
- [ ] **边界 case**想了吗？
- [ ] **异常输入**会怎样？
- [ ] **并发场景**会怎样？

### 上下游影响

- [ ] 这个测试会影响**其他测试**吗？
- [ ] 需要**特殊环境**支持吗？
- [ ] 测试后需要**恢复环境**吗？

---

## 输出前：格式检查

### 表格格式

- [ ] 步骤表格有**5 列**吗？(步骤、步骤名、操作描述、参数、预期结果)
- [ ] 步骤**连续编号**吗？(1,2,3...)
- [ ] 参数列格式统一吗？(param=value, unit)

### 验证点列表

- [ ] 有独立的**验证点列表**吗？
- [ ] 验证点**编号连续**吗？
- [ ] 验证点**可量化**吗？

### 变体场景

- [ ] 有**变体场景**说明吗？
- [ ] 变体场景的**差异点**明确吗？
- [ ] 变体场景的**预期结果**说明了吗？

---

## 设计质量自评

用以下标准给自己打分：

| 维度 | 权重 | 得分 (0-100) | 加权得分 |
|------|------|-------------|---------|
| 完整性 | 30% | | |
| 准确性 | 30% | | |
| 可执行性 | 25% | | |
| 覆盖度 | 15% | | |
| **总分** | 100% | | |

**自评结果**：
- 90-100 分：A 级，直接交付
- 70-89 分：B 级，修改后交付
- 50-69 分：C 级，需要返工
- <50 分：D 级，不可用

---

## 快速自检版 (30 秒版)

时间紧张时用这个版本：

- [ ] 有环境重置吗？
- [ ] 有验证点吗？(至少 3 个)
- [ ] 参数带单位吗？
- [ ] 有负例场景吗？
- [ ] AW 函数能映射吗？

**5 个都勾上 → 可以交付**  
**有任何一个没勾 → 补完再交**
