# 组合场景测试 Pattern

> 跨单场景的端到端组合测试。单一场景覆盖不了的真实用户场景，在这里闭环。

---

## 设计原则

1. **真实场景驱动**：不是"功能 A+ 功能 B"的简单叠加，是用户真实使用场景
2. **相互影响验证**：验证场景切换/并发时的相互影响
3. **连续性保障**：验证主业务在场景变化时的连续性

---

## CS-01: 切换中保持通话 (VoNR + A3 HO)

### 测试目的

验证 UE 在 VoNR 通话过程中执行切换时，语音业务不中断或中断时间<50ms。

### 标准步骤序列

| 步骤 | 步骤名 | 操作描述 | 参数 | 预期结果 |
|------|--------|---------|------|---------|
| 1 | 环境重置 | 清空残留配置，重置测试环境 | - | 环境干净 |
| 2 | 服务小区建立 | 建立 NR 服务小区 | cell_id=1, BW=100MHz, PCI=1 | 小区状态 ACTIVE |
| 3 | 邻区建立 | 建立同频邻区 | cell_id=2, BW=100MHz, PCI=2 | 小区状态 ACTIVE |
| 4 | 邻区配置 | 添加邻区关系 | neighbor_cell_id=2 | 邻区添加成功 |
| 5 | UE 附着 | UE 附着到服务小区 | IMSI=460001234567890 | UE 进入 CONNECTED 态 |
| 6 | IMS 注册 | UE 执行 IMS 注册 | DNN=ims | IMS 注册成功 |
| 7 | 呼叫建立 | 发起 VoNR 呼叫 | 被叫=13800138000 | 呼叫建立成功，双向语音通路 |
| 8 | 通话质量基线 | 验证初始通话质量 | duration=10s | MOS > 4.0 |
| 9 | A3 测量配置 | 配置 A3 事件测量 | threshold=-95dBm, offset=3dB | 测量配置生效 |
| 10 | 触发切换 | 移动 UE 到小区边缘 | 从 (0,0) 到 (500,0) | UE 进入切换触发区 |
| 11 | 切换中通话监控 | 监控切换过程中语音质量 | - | 中断时间<50ms 或无感知 |
| 12 | A3 MR 验证 | 验证 UE 上报 A3 测量报告 | timeout=5s | 收到 A3 MR |
| 13 | 切换命令验证 | 验证基站下发切换命令 | - | 收到 RRC Reconfiguration |
| 14 | 切换完成验证 | 验证 UE 接入目标小区 | - | 收到 RRC Reconfiguration Complete |
| 15 | 切换后通话验证 | 验证切换后通话质量 | duration=10s | MOS > 4.0 |
| 16 | 呼叫释放 | 挂断电话 | - | 正常释放 |

### 关键验证点

1. ✅ 切换过程中语音中断时间 < 50ms
2. ✅ 切换前后 MOS 分无明显下降 (下降<0.5)
3. ✅ 切换过程中无 RLC 重建
4. ✅ 切换后 RTP 通路正常

### 通过标准

| 指标 | 要求 |
|------|------|
| 语音中断时间 | < 50ms |
| MOS 分下降 | < 0.5 |
| 切换成功率 | 100% |
| 呼叫掉话率 | 0% |

| 3 | 邻区建立 | 建立同频邻区 | cell_id=2, BW=100MHz | 小区状态 ACTIVE |
| 4 | 邻区配置 | 添加邻区关系 | neighbor_cell_id=2 | 邻区添加成功 |
| 5 | UE 附着 | UE 附着到服务小区 | - | UE 进入 CONNECTED 态 |
| 6 | 启动持续 PING | 开始周期性 PING 测试 | interval=1s, count=60, dst_ip=8.8.8.8 | PING 开始执行 |
| 7 | PING 基线验证 | 验证初始 PING 时延 | 前 10 个包 | 时延<10ms，无丢包 |
| 8 | A3 测量配置 | 配置 A3 事件测量 | threshold=-95dBm, offset=3dB | 测量配置生效 |

4. ✅ 总丢包率 < 5%

## CS-03: 弱场接入测试 (Low RSRP + RACH)

### 测试目的

验证在低 RSRP 条件下，UE 仍能成功执行随机接入流程。

### 标准步骤序列

| 步骤 | 步骤名 | 操作描述 | 参数 | 预期结果 |
|------|--------|---------|------|---------|
| 1 | 环境重置 | 清空残留配置，重置测试环境 | - | 环境干净 |
| 2 | 小区建立 | 建立 NR 服务小区 | cell_id=1, BW=100MHz, power=46dBm | 小区状态 ACTIVE |
| 3 | RACH 配置验证 | 验证 SIB1 中 RACH 配置广播 | - | RACH 配置正确 |
| 4 | UE 上电 | UE 上电并搜索小区 | - | UE 搜索到小区 |
| 5 | 同步 | UE 执行 PSS/SSS 同步 | - | 同步成功 |
| 6 | MIB 获取 | UE 读取 MIB | - | MIB 获取成功 |
| 7 | SIB1 获取 | UE 读取 SIB1 | - | SIB1 获取成功 |
| 8 | 弱场配置 | 设置高 pathloss 模拟弱场 | pathloss=130dB | RSRP ≈ -90dBm |
| 9 | RSRP 验证 | 验证当前 RSRP | - | RSRP 在 -90dBm 左右 |
| 10 | Msg1 发送 | UE 发送 Preamble (可能多次重传) | - | Preamble 发送 |
| 11 | Msg2 接收 | UE 接收 RAR | timeout=50ms | 收到 RAR |
| 12 | Msg3 发送 | UE 发送 RRC Setup Request | - | Msg3 发送成功 |
| 13 | Msg4 接收 | UE 接收 RRC Setup | timeout=20ms | 收到 RRC Setup |
| 14 | 接入完成验证 | UE 进入 RRC CONNECTED 态 | - | 状态=CONNECTED |
| 15 | 接入时延统计 | 统计从 Msg1 到接入完成的时延 | - | 时延<500ms |

### 关键验证点

1. ✅ 弱场条件下 RACH 成功率 > 95%
2. ✅ 接入时延 < 500ms
3. ✅ Preamble 重传次数 < 5 次
4. ✅ 接入后上行功率在合理范围

### 通过标准

| 指标 | 要求 |
|------|------|
| RSRP 范围 | -85 ~ -95 dBm |
| 接入成功率 | > 95% |
| 接入时延 | < 500ms |
| Preamble 重传 | < 5 次 |

## 组合场景设计检查清单

设计组合场景时需检查：

- [ ] 是否有真实的用户场景驱动？(不是为组合而组合)
- [ ] 主业务是什么？连续性如何保障？
- [ ] 副业务是什么？对主业务的影响？
- [ ] 场景切换点在哪里？如何触发？
- [ ] 验证指标是否可量化？
- [ ] 通过标准是否明确？
- [ ] 是否有对应的单场景 Pattern 作为基础？
