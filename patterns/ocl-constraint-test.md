# OCL 参数约束测试 Pattern

## 触发关键词

- 中文：OCL、参数约束、配置校验、操作维护、OMC
- 英文：OCL、parameter constraint、configuration validation、OMC

## 测试目的

验证 OCL (Operation and Maintenance Channel Link) 通道下，基站参数配置的正确性约束检查能力，包括参数范围、依赖关系、互斥条件等校验规则。

## 协议参考

- 3GPP TS 32.500: 电信管理 - 配置管理
- 3GPP TS 28.400: 电信管理 - 通用架构
- 华为基站参数配置规范

## OCL 参数约束类型

| 约束类型 | 说明 | 示例 |
|---------|------|------|
| 范围约束 | 参数值必须在指定范围内 | PCI: 0-1007 |
| 类型约束 | 参数必须是特定数据类型 | cell_id: integer |
| 依赖约束 | 参数生效依赖其他参数配置 | 邻区频点依赖频点配置 |
| 互斥约束 | 某些参数不能同时生效 | 同频点不能配置冲突 PCI |
| 条件约束 | 满足条件时参数才有约束 | 带宽=200MHz 时 SCS 只能 30/60 |
| 数量约束 | 配置项数量限制 | 邻区数≤32 |

## 标准步骤序列 (参数约束校验)

| 步骤 | 步骤名 | 操作描述 | 参数 | 预期结果 |
|------|--------|---------|------|---------|
| 1 | 环境重置 | 清空残留配置，重置测试环境 | - | 环境干净 |
| 2 | OCL 连接建立 | 建立 OCL 会话 | ocl_address=192.168.1.1 | 连接成功 |
| 3 | OCL 状态验证 | 验证 OCL 通道状态 | - | 状态=CONNECTED |
| 4 | 正常参数配置 | 配置合法参数 | 参数在约束范围内 | 配置成功 |
| 5 | 参数约束验证 | 验证配置已生效 | - | 参数值正确 |
| 6 | 越界参数配置 | 尝试配置越界参数 | 参数超出范围 | 配置失败，返回错误码 |
| 7 | 错误码验证 | 验证返回的约束错误码 | - | 错误码=PARAM_OUT_OF_RANGE |
| 8 | 错误信息验证 | 验证错误信息描述 | - | 错误信息包含参数名和范围 |
| 9 | 依赖约束测试 | 配置违反依赖约束的参数 | - | 配置失败 |
| 10 | 互斥约束测试 | 配置违反互斥约束的参数 | - | 配置失败 |
| 11 | 恢复验证 | 用合法参数配置 | - | 配置成功 |

## 关键参数约束规则

### 小区级参数约束

| 参数 | 约束类型 | 约束规则 | 错误码 |
|------|---------|---------|--------|
| cell_id | 范围约束 | 0-255 | OUT_OF_RANGE |
| pci | 范围约束 | 0-1007 | OUT_OF_RANGE |
| bandwidth | 枚举约束 | 50/100/200 MHz | INVALID_VALUE |
| scs | 条件约束 | 带宽=50→15/30; 带宽=100/200→30/60 | INVALID_COMBINATION |
| arfcn | 范围约束 | 627566-638000 (n78) | OUT_OF_RANGE |
| power | 范围约束 | 0-60 dBm | OUT_OF_RANGE |
| cell_name | 长度约束 | ≤64 字符 | STRING_TOO_LONG |

### 邻区参数约束

| 参数 | 约束类型 | 约束规则 | 错误码 |
|------|---------|---------|--------|
| neighbor_cell_id | 范围约束 | 0-255, 不能与 cell_id 相同 | OUT_OF_RANGE/SELF_LOOP |
| nci | 范围约束 | 0-68719476735 | OUT_OF_RANGE |
| qoffset | 范围约束 | -24~24 dB | OUT_OF_RANGE |
| 邻区数量 | 数量约束 | ≤32 (同频), ≤16 (异频) | MAX_COUNT_EXCEEDED |

### 测量参数约束

| 参数 | 约束类型 | 约束规则 | 错误码 |
|------|---------|---------|--------|
| threshold | 范围约束 | -140~-44 dBm | OUT_OF_RANGE |
| hysteresis | 范围约束 | 0-30 dB | OUT_OF_RANGE |
| ttt | 枚举约束 | 0/40/64/80/100/128/160/240/320/480/640/1024/1280/2560/5120 ms | INVALID_VALUE |
| offset | 范围约束 | -24~24 dB | OUT_OF_RANGE |

## 通过标准

1. ✅ OCL 连接建立成功
2. ✅ 合法参数配置成功并生效
3. ✅ 越界参数配置被正确拒绝
4. ✅ 返回正确的错误码和错误信息
5. ✅ 依赖约束校验正确
6. ✅ 互斥约束校验正确
7. ✅ 配置失败后系统状态一致

## 常见失败原因

| 现象 | 可能原因 | 排查方法 |
|------|---------|---------|
| 约束校验未触发 | 约束规则未启用 | 检查约束配置开关 |
| 错误码不正确 | 约束规则定义错误 | 检查约束规则定义 |
| 错误信息模糊 | 错误模板配置问题 | 检查错误消息模板 |
| 部分约束无效 | 约束规则覆盖不全 | 补充约束规则 |

## 约束规则检查清单

配置约束规则时需检查：

- [ ] 范围约束：最小值、最大值、是否包含边界
- [ ] 类型约束：integer、string、enum、boolean
- [ ] 枚举约束：所有合法值的完整列表
- [ ] 条件约束：条件表达式、依赖参数
- [ ] 依赖约束：依赖的参数或配置项
- [ ] 互斥约束：互斥的参数组合
- [ ] 数量约束：最大数量、是否可配置
- [ ] 长度约束：字符串最大/最小长度
- [ ] 格式约束：正则表达式匹配规则
- [ ] 错误码：每种约束对应的错误码
- [ ] 错误信息：用户友好的错误描述

## OCL 约束测试自动化建议

```python
# 约束测试伪代码结构

class TestOCLConstraints(unittest.TestCase):
    
    def test_parameter_range_constraints(self):
        """参数范围约束测试"""
        # 遍历所有范围约束参数
        for param in RANGE_CONSTRAINT_PARAMS:
            # 测试下限 -1
            result = configure(param, param.min - 1)
            self.assertFalse(result.success)
            self.assertEqual(result.error_code, "OUT_OF_RANGE")
            
            # 测试上限 +1
            result = configure(param, param.max + 1)
            self.assertFalse(result.success)
            self.assertEqual(result.error_code, "OUT_OF_RANGE")
    
    def test_condition_constraints(self):
        """条件约束测试"""
        # 遍历所有条件约束组合
        for combo in CONDITION_CONSTRAINTS:
            result = configure(combo.param1, combo.value1, 
                             combo.param2, combo.value2)
            if combo.is_valid:
                self.assertTrue(result.success)
            else:
                self.assertFalse(result.success)
                self.assertEqual(result.error_code, "INVALID_COMBINATION")

```
