# 邻区配置测试 Pattern

## 触发关键词

- 中文：邻区、neighbor、Xn、切换准备
- 英文：neighbor、Xn setup、handover preparation

## 测试目的

验证邻区关系正确配置后，能够支持切换流程的正常执行。

## 协议参考

- 3GPP TS 38.423: Xn 应用协议
- 3GPP TS 38.300: NR 总体描述

## 邻区配置关键参数

| 参数 | 说明 | 取值范围 |
|------|------|---------|
| cell_id | 服务小区 ID | 0-255 |
| neighbor_cell_id | 邻区 ID | 0-255 |
| nci | NR Cell Identity | 0-68719476735 |
| pci | 邻区 PCI | 0-1007 |
| freq | 邻区频点 | 0-2016666 |
| qoffset | 小区偏移 | -24~24 dB |

## 标准步骤序列 (同频邻区配置)

| 步骤 | 步骤名 | 操作描述 | 参数 | 预期结果 |
|------|--------|---------|------|---------|
| 1 | 环境重置 | 清空残留配置，重置测试环境 | - | 环境干净 |
| 2 | 服务小区建立 | 建立 NR 服务小区 | cell_id=1, BW=100MHz, SCS=30kHz, arfcn=629166, PCI=1 | 小区状态 ACTIVE |
| 3 | 邻区建立 | 建立同频邻区 | cell_id=2, BW=100MHz, SCS=30kHz, arfcn=629166, PCI=2 | 小区状态 ACTIVE |
| 4 | 邻区关系添加 | 配置邻区关系 | neighbor_cell_id=2, nci=2, pci=2, freq=629166, qoffset=0 | 添加成功 |
| 5 | 邻区列表验证 | 查询邻区列表确认配置 | cell_id=1 | 邻区列表中包含 cell_id=2 |
| 6 | 邻区状态验证 | 验证邻区状态正常 | neighbor_cell_id=2 | 状态=ACTIVE |

## 关键参数推荐值

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 同频邻区 freq | 与服务小区相同 | Intra-frequency |
| 异频邻区 freq | 不同频点 | Inter-frequency |
| qoffset | 0 | 默认无偏移 |
| nci | cell_id 扩展 | 如 cell_id=1 → nci=1 |

## 通过标准

1. ✅ 邻区添加命令返回成功
2. ✅ 邻区列表查询能获取到配置的邻区
3. ✅ 邻区状态正常
4. ✅ 能够基于邻区关系触发切换

## 常见失败原因

| 现象 | 可能原因 | 排查方法 |
|------|---------|---------|
| 邻区添加失败 | 邻区 ID 冲突 | 检查是否已存在相同邻区 |
| 邻区状态异常 | 目标小区未建立 | 先建立邻区小区 |
| 切换失败 | Xn 链路未通 | 检查 Xn 配置 |
| PCI 混淆 | 多个邻区同 PCI | 确保邻区 PCI 唯一 |
