# VoNR 呼叫测试 Pattern

## 触发关键词

- 中文：VoNR、语音、呼叫、通话、VoLTE
- 英文：VoNR、voice call、VoLTE、EPS fallback

## 测试目的

验证 5G 网络下 VoNR 语音呼叫功能正常，包括呼叫建立、通话质量、呼叫释放等全流程。

## 协议参考

- 3GPP TS 24.501: 5G MM 协议
- 3GPP TS 24.605: 5G CC 协议
- 3GPP TS 26.114: IMS 语音和语音编解码

## 关键 KPI 要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 呼叫建立时延 | < 3s | 从 INVITE 到 200 OK |
| 呼叫建立成功率 | > 99% | 正常条件下的成功率 |
| MOS 分 | > 4.0 | 语音质量评分 |
| 掉话率 | < 0.1% | 通话中异常释放概率 |

## 标准步骤序列 (VoNR 主叫)

| 步骤 | 步骤名 | 操作描述 | 参数 | 预期结果 |
|------|--------|---------|------|---------|
| 1 | 环境重置 | 清空残留配置，重置测试环境 | - | 环境干净 |
| 2 | 小区建立 | 建立 NR 服务小区 | cell_id=1, BW=100MHz, SCS=30kHz | 小区状态 ACTIVE |
| 3 | UE 附着 | UE 附着到网络 | IMSI=460001234567890 | UE 进入 CONNECTED 态 |
| 4 | IMS 注册 | UE 执行 IMS 注册 | P-CSCF 地址配置 | IMS 注册成功 |
| 5 | IMS 状态验证 | 验证 UE IMS 注册状态 | - | IMS 状态=REGISTERED |
| 6 | 发起呼叫 | 主叫 UE 发起 INVITE 消息 | 被叫号码=13800138000 | INVITE 发送成功 |
| 7 | 呼叫建立验证 | 验证被叫响应 200 OK | timeout=10s | 收到 200 OK |
| 8 | 通话建立验证 | 验证双向语音通路建立 | - | 双方可通话 |
| 9 | 通话质量验证 | 验证通话过程中的语音质量 | duration=30s | MOS > 4.0, 无断续 |
| 10 | 呼叫释放 | 主叫挂断电话 | - | BYE 消息发送成功 |
| 11 | 释放完成验证 | 验证呼叫正常释放 | - | 双方回到空闲态 |

## 关键参数推荐值

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 编解码 | EVS-WB | 增强语音服务宽带 |
| 呼叫时长 | 30-60s | 单次通话测试时长 |
| P-CSCF 地址 | 192.168.1.1 | IMS 核心网地址 |
| DNN | ims | IMS 专用 DNN |

## 通过标准

1. ✅ IMS 注册成功
2. ✅ 呼叫建立时延 < 3s
3. ✅ 呼叫建立成功率 100%
4. ✅ 通话过程中 MOS 分 > 4.0
5. ✅ 通话无断续、无杂音
6. ✅ 呼叫正常释放，无异常

## 常见失败原因

| 现象 | 可能原因 | 排查方法 |
|------|---------|---------|
| IMS 注册失败 | DNN 配置错误 | 检查 PDU Session DNN=ims |
| 呼叫建立失败 | 被叫不可达 | 检查被叫 IMS 状态 |
| 单向语音 | RTP 通路单向 | 检查 UPF 配置 |
| 通话断续 | QoS 保障不足 | 检查 5QI=1 的 GBR 配置 |
| 掉话 | 切换中 QoS 丢失 | 检查切换时的 QoS Flow 迁移 |
