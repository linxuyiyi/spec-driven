# 5G 基站自动化测试 - AI 辅助测试技能包

## 项目概述

这是一个基于 opencode skill 构建的 5G 基站自动化测试辅助工具，能够将模糊的测试意图扩写为详细的测试步骤描述。

## 项目结构

```
5g-test-skill/
├── SKILL.md                      # 主指令文件 (AI 行为定义)
├── README.md                     # 项目说明 (v1.2.0)
├── patterns/                     # 测试场景模式库 (11 个场景 + 索引 + 组合场景)
│   ├── INDEX.md                  # 场景索引 + 关键词速查
│   ├── composite-scenarios.md    # 5 个组合场景
│   ├── handover-a3.md            # A3 同频切换测试
│   ├── handover-a5.md            # A5 异频切换测试
│   ├── measurement-rrm.md        # RRM 测量报告测试
│   ├── cell-setup.md             # 小区建立测试
│   ├── neighbor-config.md        # 邻区配置测试
│   ├── service-test.md           # 业务测试 (PING/FTP)
│   ├── vonr-call.md              # VoNR 呼叫测试
│   ├── csi-measurement.md        # CSI 测量上报测试
│   ├── rach-access.md            # 随机接入测试
│   ├── negative-test.md          # 负例测试通用模式
│   └── ocl-constraint-test.md    # OCL 参数约束测试
└── references/
    ├── aw-function-catalog.md    # AW 测试函数目录 (含使用位置索引)
    ├── quality-gate.md           # 质量验收标准 + Badcase 模板
    ├── quick-reference.md        # 快速参考手册
    └── test-design-checklist.md  # 测试设计检查清单
```

## 使用方式

### 基本用法

用户输入测试意图，AI 输出详细的测试步骤描述。

**示例输入**:
```
写一个 A3 切换的测试用例
```

**示例输出**:
```markdown
## 测试步骤

| 步骤 | 步骤名 | 操作描述 | 参数 | 预期结果 |
|------|--------|---------|------|---------|
| 1 | 环境重置 | 清空残留配置，重置测试环境 | - | 环境干净 |
| 2 | 服务小区建立 | 建立 NR 服务小区 | cell_id=1, BW=100MHz... | 小区激活 |
| 3 | 邻区配置 | 添加同频邻区关系 | neighbor_cell_id=2... | 邻区添加成功 |
...

## 验证点
1. UE 在满足 A3 条件后 240ms+5s 内上报测量报告
2. 测量报告中包含邻区 RSRP 且优于服务小区 3dB 以上
...
```

### 支持的测试场景

| 场景类别 | 覆盖的测试类型 |
|---------|--------------|
| 切换测试 | A3 同频切换、A5 异频切换 |
| 测量测试 | RRM 测量 (A1-A6)、CSI 测量上报 |
| 小区管理 | 小区建立/删除/修改、邻区配置 |
| 接入测试 | 随机接入 (RACH)、波束失败恢复 |
| 业务测试 | PING、FTP 吞吐量、VoNR 呼叫 |
| 负例测试 | 参数非法、状态不符、资源不足等 |

## 输出格式说明

### 测试步骤表格

| 列名 | 说明 |
|------|------|
| 步骤 | 步骤序号 |
| 步骤名 | 动词 + 名词格式 (如"小区建立") |
| 操作描述 | 清晰说明做什么 |
| 参数 | 参数名=参数值，带单位 |
| 预期结果 | 步骤执行后应达到的状态 |

### 验证点列表

独立列出所有需要验证的检查点，便于审核测试覆盖度。

## 参数默认值

| 参数 | 默认值 | 说明 |
|------|--------|------|
| cell_id | 1 | 服务小区 ID |
| neighbor_cell_id | 2 | 邻区 ID |
| bandwidth | 100 | 带宽 MHz |
| scs | 30 | 子载波间隔 kHz |
| threshold (A3) | -95 | 门限 dBm |
| hysteresis | 3 | 迟滞 dB |
| ttt | 240 | 触发时间 ms |
| timeout | 5 | 验证超时秒数 |

## 落地实施步骤

### 步骤 1: 梳理 AW 函数 (回公司后)

```bash
# 提取所有 aw_ 开头的函数
grep -rn "^def aw_" <测试代码目录> --include="*.py"

# 提取函数签名
grep -A5 "^def aw_" <目录> --include="*.py"
```

整理后更新 `references/aw-function-catalog.md`。

### 步骤 2: 验证 Skill 效果

用 3-5 个真实测试需求测试：
- "写一个 A3 切换测试用例"
- "配置一个 A5 事件测量，门限 -100 和 -90"
- "验证小区建立流程"

### 步骤 3: 补充 Pattern

根据实际测试场景，从 SKILL.md 的映射表中扩展新的 pattern 文件。

### 步骤 4: 集成到工作流

将 skill 集成到日常测试设计流程中：
1. 测试需求分析
2. **AI 辅助步骤扩写** ← 本工具
3. 人工审核步骤
4. 测试脚本编写
5. 测试执行
