# OCL 约束测试设计知识

## 一、约束分类

### 1. 条件约束（implies）

形式：

```
A implies B
```

测试点：

| 场景 | A | B | 结果 |
|------|---|---|------|
| 正例 1 | 真 | 真 | 成功 |
| 正例 2 | 假 | 任意 | 成功 |
| 反例 | 真 | 假 | 失败 |

---

### 2. AND 约束

形式：

```
A and B
```

测试点：
- A=true, B=true → 成功
- A=false → 失败
- B=false → 失败

---

### 3. OR 约束

形式：

```
A or B
```

测试点：
- A=false, B=false → 失败
- 其他 → 成功

---

### 4. NOT 约束

形式：

```
not A
```

测试点：
- A=true → 失败
- A=false → 成功

---

### 5. 互斥约束

形式：

```
not (A and B)
```

测试点：
- A=true, B=true → 失败
- A=true, B=false → 成功
- A=false, B=true → 成功
- A=false, B=false → 成功

---

## 二、测试生成方法

### 1. 约束拆解

将复杂表达式拆分：

```ocl
(A and B) implies C
```

拆为：
- A and B
- implies

---

### 2. 真值表驱动

基于逻辑生成：
- 所有关键组合
- 最小覆盖集

---

### 3. 边界增强

例如：

```ocl
prb <= 273
```

增加：
- 273（边界）
- 274（越界）

---

## 三、典型错误

1. 条件未触发仍校验
2. 条件触发但未限制
3. 错误未阻止配置
4. 错误未返回告警

---

## 四、约束覆盖率

| 指标 | 说明 |
|------|------|
| 约束覆盖率 | 已验证约束数 / 总约束数 |
| 分支覆盖 | 已覆盖分支数 / 总分支数 |
| 边界覆盖 | 已测试边界点数 / 总边界点数 |
