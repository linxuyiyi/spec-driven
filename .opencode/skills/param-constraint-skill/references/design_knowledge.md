# 参数约束测试设计知识

## 测试维度
- **单参数校验**：数据类型（整型、字符、枚举）、取值范围（最大、最小、步长）、空值/默认值处理。
- **多参数联动**：参数间互斥（如特性 A 和特性 B 不能同时开启）、参数间依赖（配置参数 B 的前提是参数 A 必须为特定值）。
- **生效机制**：立即生效参数、重启生效参数的提示与状态机转移。

## 关键测试点与边界值提取规则
1. **边界值测试 (BVA)**：
   - 提取规则：设有效范围为 `[Min, Max]`，需测试 `Min`, `Max`, `Min-1`, `Max+1`。
   - 预期行为：`Min`与`Max`下发成功；`Min-1`与`Max+1`下发失败，且网管返回明确的越界错误码。
2. **非法枚举/格式测试**：
   - 提取规则：输入不在枚举列表中的值，或输入包含特殊字符的非法字符串。
   - 预期行为：语法解析层拦截，拒绝下发。
3. **互斥参数校验 (Mutex)**：
   - 场景：特性开关 A = ON 时，特性开关 B 必须为 OFF。
   - 测试点：先下发 A=ON，再尝试下发 B=ON。
   - 预期行为：第二条命令被拒绝，提示业务冲突。
4. **依赖关系校验 (Dependency)**：
   - 场景：配置对象的子节点时，父节点或必填前置参数缺失。
   - 预期行为：提示"依赖对象不存在"或"缺失必填参数"。
