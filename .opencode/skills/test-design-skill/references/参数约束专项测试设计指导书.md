## 1. 专项说明

### 1.1 专项定义
参数约束专项聚焦 5G 基站配置参数（MO/Parameter）的约束校验测试，验证参数配置符合约束规则，系统正确拒绝非法配置。

### 1.2 适用范围
- 适用系统：5G 基站 OAM 配置系统
- 适用阶段：设计阶段、需求分析阶段
- 适用对象：配置参数、开关参数、Bit字段

### 1.3 专项特点
- 约束规则形式多样：OCL 表达式、文字描述、配置规则
- 需理解三要素：触发条件、约束对象、检查时机
- 边界清晰：有效/无效配置结果明确

## 2. 测试设计思路

### 2.1 设计原则
- **约束优先**：优先覆盖约束规则的触发条件和违反场景
- **三要素分析**：每个约束规则必须识别 Trigger / Target / Check Point
- **分支覆盖**：正例、反例、修正、无关四种分支全覆盖

### 2.2 设计流程

| 步骤 | 活动 | 产出 |
|------|------|------|
| 1 | 约束规则分析 | 识别 Trigger / Target / Check Point |
| 2 | 分支场景提取 | 正例/反例/修正/无关场景列表 |
| 3 | 测试数据设计 | 有效值、无效值、边界值 |
| 4 | 用例编写 | 按用例要素填写 |

### 2.3 风险识别与场景筛选

**常见风险点**：
- 约束规则解析错误（OCL 理解偏差）
- 边界值遗漏（Min/Max 临界点）
- 多参数联动时序问题

### 2.4 常用设计方法

| 方法 | 适用场景 | 说明 |
|------|----------|------|
| 三要素分析 | 所有约束规则 | 识别 Trigger/Target/Check Point |
| BVA 边界值 | 取值范围参数 | Min/Max/Min-1/Max+1 |
| 等价类划分 | 枚举类型参数 | 有效枚举/无效枚举 |
| 正交试验 | 多参数组合 | 参数组合覆盖 |

### 2.5 设计思路示例

**示例：Bit字段开关约束**

约束规则：`ASwitch.bitAnd(4) = 4 implies TxRxMode ∈ {3, 4, 13}`

| 三要素 | 内容 |
|--------|------|
| Trigger | ASwitch.bit4 = 1（B_SW 开启） |
| Target | TxRxMode |
| Check Point | 配置修改时检查 |

**分支场景**：
- 正例：B_SW开 + TxRxMode=32T32R → 成功
- 反例：B_SW开 + TxRxMode=2T2R → 失败
- 修正：B_SW开 + TxRxMode先改32T32R再开B_SW → 成功
- 无关：B_SW关 + TxRxMode任意 → 成功

## 3. 测试用例规范

### 3.1 用例要素

| 字段 | 说明 | 示例 |
|------|------|------|
| 用例编号 | 唯一标识，规则 `{MO}-{类型}-{序号}` | NRDUCell-FUNC-001 |
| 用例名称 | 【参数】场景，≤30字 | 【发射功率】有效值200配置成功 |
| 用例级别 | P0/P1/P2 | P0为核心约束必检 |
| 前置条件 | 基站状态、网管连接、参数初始值 | 小区已建立 |
| 测试步骤 | 配置命令 + 提交 + 查询验证 | 1.配置NRDUCellTxPower=200 |
| 预期结果 | 成功/失败及错误原因 | 配置成功，查询返回200 |
| 后置条件 | 恢复原值或删除临时配置 | 恢复默认 |

### 3.2 编写要求
- 步骤为原子操作，可独立执行
- 预期结果需说明错误原因，不只是"失败"
- 互斥类用例需测试顺序和同时下发两种场景

### 3.3 用例组织
按约束类型分组：单参数边界 / 枚举校验 / 互斥 / 依赖 / Bit字段

## 7. 附录

### 7.1 术语表

| 术语 | 解释 |
|------|------|
| OCL | Object Constraint Language，对象约束语言 |
| Trigger | 触发条件，约束生效的前提 |
| Target | 约束对象，被约束的参数 |
| Check Point | 检查时机，配置/创建/激活时检查 |
| Bit字段 | 通过位掩码表示的开关组合 |

### 7.2 参考标准与文献
- 3GPP TS 28.541 (5G NRM)
- 基站设备商 MML 命令参考手册

### 7.3 用例示例

#### 示例1：单参数边界

**测试对象**：`NRDUCellTxPower`（基站发射功率）

**约束规则**：取值范围 [0, 200]，单位 dBm

| 用例编号 | 用例名称 | 用例级别 | 前置条件 | 测试步骤 | 预期结果 | 后置条件 |
|----------|----------|----------|----------|----------|----------|----------|
| NRDUCell-FUNC-001 | 【发射功率】有效值0配置成功 | P0 | 小区已建立 | 1.配置NRDUCellTxPower=0<br>2.提交<br>3.查询配置 | 配置成功，查询返回0 | 恢复默认 |
| NRDUCell-NEG-001 | 【发射功率】越界值-1被拒绝 | P0 | 小区已建立 | 1.配置NRDUCellTxPower=-1<br>2.提交 | 配置失败，提示有效范围[0,200] | 无 |

#### 示例2：多参数互斥

**测试对象**：`FeatureA_SW` 与 `FeatureB_SW`

**约束规则**：两个特性开关互斥，不能同时为 ON

| 用例编号 | 用例名称 | 用例级别 | 前置条件 | 测试步骤 | 预期结果 | 后置条件 |
|----------|----------|----------|----------|----------|----------|----------|
| MUTEX-NEG-001 | 【互斥】先开A再开B被拒绝 | P0 | A=OFF, B=OFF | 1.开启FeatureA_SW<br>2.提交<br>3.开启FeatureB_SW<br>4.提交 | 第3步失败，提示互斥 | 关闭A |
| MUTEX-FUNC-001 | 【互斥】单独开A成功 | P0 | A=OFF, B=OFF | 1.开启FeatureA_SW<br>2.提交 | 配置成功 | 关闭A |
