# JWT 认证方案选型

## 背景

DRF-Vue3 项目需要实现用户认证机制。

## 目标

- [ ] 实现无状态认证
- [ ] 支持跨域请求
- [ ] 移动端友好
- [ ] 支持 token 刷新

## 可选方案

### 方案 A：Session 认证

**优点:**
- Django 内置，简单易用
- 安全性高

**缺点:**
- 需要服务端存储 session
- 跨域配置复杂
- 移动端不友好

### 方案 B：JWT (SimpleJWT)

**优点:**
- 无状态，服务端不存储
- 跨域友好
- 移动端友好
- 支持 token 刷新

**缺点:**
- token 被窃取后无法撤销（直到过期）
- token 长度较长

### 方案 C：OAuth2

**优点:**
- 标准协议
- 支持第三方认证

**缺点:**
- 实现复杂
- 过度设计

## 最终决策

**选择方案**: 方案 B - JWT (SimpleJWT)

## 决策理由

1. 项目需要 API 驱动，无状态认证更合适
2. Vue3 前端和后端分离部署，跨域是必须的
3. SimpleJWT 提供 token 刷新机制，平衡安全性和便利性

## 影响

### 积极影响

- 简化服务端部署（无需 session 存储）
- 前后端认证流程统一
- 易于扩展到移动端

### 潜在风险

- token 泄露风险需要通过 HTTPS 和短有效期缓解
- 需要实现 token 黑名单机制处理注销
